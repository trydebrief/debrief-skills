# /debrief cos

Bundled flow for a Chief of Staff: exec brief, 1:1 prep for the principal's key reports, planning-cycle tracker, cross-team status.

## Apps
- **Granola** — meetings the principal missed; recent 1:1s.
- **Slack** — escalations and decisions awaiting input. Also check saved-for-later via `is:saved`.
- **Gmail** — escalations and decisions pending.
- **Linear** — milestones and cross-team status.
- **Notion** — planning-cycle tracker and the CoS working doc.

## Step 1: Confirm which pieces to run
Default: all four. User can subset.

## Step 2: Run in sequence
1. **Exec brief** — for the principal (run `/debrief exec` instructions)
2. **1:1 prep** — for today's 1:1s on the principal's calendar (run `/debrief oneone`)
3. **Planning-cycle tracker** — pull current cycle status from Notion
4. **Cross-team status** — aggregate from Slack team channels and the Linear project list; include Slack `is:saved`

## Step 3: Bundle
Single document, each section clearly headed. Offer to save to the CoS working doc in Notion.

## Guidelines
- This is a lot. Don't run it daily — weekly or pre-exec-meeting.
- Each section should stand alone; the principal may read only one.
