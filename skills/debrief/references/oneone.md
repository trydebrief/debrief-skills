# /debrief oneone

Prep for a 1:1: last meeting's notes, recent work from that person, suggested topics.

## Apps
- **Granola** — last 1:1 transcript / notes.
- **Notion** — written 1:1 notes if stored there.
- **Linear** — the person's recent ticket activity.
- **Slack** — their recent public-channel messages. Also check saved-for-later via `is:saved`.

## Step 1: Identify the person
From the next calendar event, or user specifies.

## Step 2: Pull context
- Use Granola for the last 1:1's transcript.
- Use Notion for any written 1:1 notes.
- Use Linear for the person's recent ticket activity.
- Use Slack for their recent public-channel messages (NOT DMs); include `is:saved` items the user bookmarked about them.

## Step 3: Build the brief
- **Open threads from last 1:1** (what was promised, what was asked)
- **What they've been working on** (Linear, Slack)
- **What's changed in their world** (new project, shipped thing, notable Slack moment)
- **Suggested topics** — 2–3 grounded in the above

## Guidelines
- Don't surveil. Public Slack + work artifacts only. Skip DMs.
- Suggested topics should be questions, not statements.
