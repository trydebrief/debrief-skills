# /debrief sayno

Meetings and requests the user should decline this week, with reasoning.

## Apps
- **Google Calendar** — find optional/thin-agenda invites and recurring meetings.
- **Gmail** — find inbound asks not tied to priorities.
- **Slack** — find channel asks/requests not tied to priorities. Also check saved-for-later via `is:saved`.

## Step 1: Scan
- Use Google Calendar for invites where user is optional or agenda is thin, plus recurring meetings with low recent value.
- Use Gmail to find inbound asks not tied to a priority.
- Use Slack to find channel asks/requests not tied to a priority; include `is:saved` items.

## Step 2: Propose declines
For each:
- What it is (link to source)
- Why decline (no clear ask, not tied to priorities, someone else can cover)
- Draft decline note (polite, specific, offers alternative if relevant)

## Step 3: Confirm
Show list. User picks which to decline. Send decline notes via Gmail draft / Slack draft only after confirmation.

## Guidelines
- Tone: warm but firm. "I'm going to skip this one to protect focus time" beats "I can't make it".
- Don't propose declines for anything with the user's manager or a key external contact without explicit user sign-off.
