# /debrief premtg

For any calendar invite: who's attending, recent context, what they'll likely want.

## Apps
- **Google Calendar** — invite details, attendees, agenda.
- **Gmail** — recent threads with attendees.
- **Granola** — past meetings with the same group.
- **Slack** — recent shared activity with attendees. Also check saved-for-later via `is:saved`.

## Step 1: Pull the invite
Use Google Calendar for the target event (next one, or user specifies).

## Step 2: Per attendee
For each attendee:
- Role / relationship
- Last interaction — use Gmail to find the last email thread, Granola for the last meeting together, Slack for the last shared thread (include Slack `is:saved`)
- Any open threads with them

## Step 3: Read the room
Based on the invite description, attendee mix, and recent context:
- What they probably want from this meeting
- What the user should be ready to answer
- Any landmines (recent tension in Slack/Gmail, unresolved issue)

## Step 4: Present
Keep under 300 words. Lead with "what they want".

## Guidelines
- If the meeting has no description and no recent context, say so — don't pad with generic observations.
