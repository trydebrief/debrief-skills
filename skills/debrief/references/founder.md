# /debrief founder

Investor update, fundraising follow-ups, hiring pipeline check.

## Apps
- **Gmail** — investor threads and prior updates.
- **HubSpot** — fundraising pipeline and hiring pipeline.
- **Notion** — metrics and prior update format.
- **Slack** — hiring decisions awaiting input. Also check saved-for-later via `is:saved`.

## Step 1: Investor update
Run the `/debrief board` instructions if it's near month-end, else skip.

## Step 2: Fundraising follow-ups
- Use HubSpot for investors in active conversation
- Use Gmail to confirm last touch and promised next steps
- Flag overdue commitments from either side

## Step 3: Hiring pipeline
- Use HubSpot for open roles and candidates in active stages
- Flag stalled candidates (>7 days no movement)
- Use Slack to find decisions pending from the user (include `is:saved`)

## Step 4: Present as one briefing
Clearly sectioned. End with a "today's 1–3 things" recommendation.

## Guidelines
- Fundraising data is sensitive. Never post to Slack; keep in Notion or Gmail draft only.
