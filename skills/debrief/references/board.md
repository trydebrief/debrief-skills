# /debrief board

Monthly investor update, pulling metrics and narrative.

## Apps
- **Notion** — agreed-upon metrics tracker and prior update format.
- **Linear** — what shipped this month (product).
- **Gmail** — investor relationships and prior updates for tone-matching.

## Step 1: Gather metrics
Use Notion to pull the agreed-upon metrics page (ARR, growth, pipeline, headcount, runway). Confirm values with user before drafting.

## Step 2: Gather narrative
- Use Linear for shipped this month (product side).
- Use Notion for GTM updates if tracked there.
- Use Gmail to find prior investor updates for tone-matching.

Cover:
- Shipped this month (product + GTM)
- Wins (customers, hires, partnerships)
- Challenges (honest, not sandbagged)
- Asks (introductions, advice, capital)

## Step 3: Draft
Match the user's prior update format from Gmail/Notion if one exists. Otherwise default to: Metrics → Highlights → Challenges → Asks.

## Step 4: Review gate
Always show the full draft. Never send via Gmail. Investor updates are high-stakes and must be reviewed line-by-line.

## Guidelines
- Numbers must be sourced. Never estimate.
- Challenges section builds trust. A too-rosy update is a yellow flag.
