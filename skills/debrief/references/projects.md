# /debrief projects

Status of each active initiative: owner, state, last update.

## Apps
- **Linear** — active projects, owners, target dates, ticket-level activity.
- **Notion** — initiative pages and last substantive edits.

## Step 1: Pull active projects
- Use Linear to list active/in-progress projects with owners and target dates.
- Use Notion to find linked initiative pages.

## Step 2: Per project
- Owner (Linear)
- Target date (Linear, if set)
- Last meaningful update (date + who) — check both Linear ticket movement and Notion page edits
- Current state: on track / at risk / slipping / stalled
- One-line summary of what's next

## Step 3: Present as a table
Sortable by risk level. Lead with at-risk and stalled. Link each row to the Linear project and Notion page.

## Guidelines
- "Last update" = last substantive activity, not last edit timestamp. A comment that says "still working on it" is not an update.
