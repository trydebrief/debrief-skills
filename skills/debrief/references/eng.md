# /debrief eng

PR review queue, design-review prep, epic progress tracker.

## Apps
- **GitHub** — open PRs awaiting the user's review.
- **Linear** — epic progress and the user's owned projects.
- **Notion** — design docs and proposals.

## Step 1: PR queue
Use GitHub to list open PRs awaiting the user's review. Sort by age. Flag any tagged urgent.

## Step 2: Design review prep
For any design reviews on the calendar today/tomorrow, use Notion to pull the linked doc, summarize the proposal, and suggest questions based on open epics in Linear.

## Step 3: Epic progress
Use Linear for epics the user owns: status, last activity, blockers, target date.

## Step 4: Present
Three sections. Lead with PR queue (lowest effort, most frequent).

## Guidelines
- PR review suggestions shouldn't replace reading the code. Flag which PRs look low-risk (docs, config, small refactors) vs. which need careful review.
